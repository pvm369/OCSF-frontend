Deployment Guide

GET IT
RUNNING

Follow the steps below exactly as specified in the README. Each step is independently copyable — click any code block to copy the command.

Python 3.9+ Java 8 or 11+ Git pip ~5 min setup
01
PREREQUISITES
Ensure your environment has these before continuing
⚠️
Java is required by PySpark. Without Java 8 or 11+, PySpark will fail to initialize even if Python dependencies install correctly.
  • Python 3.9+ — Check with python --version
  • Java 8 or 11+ — Check with java -version. Install from adoptium.net if missing.
  • pip — Should come with Python. Check with pip --version
  • Git — Required for cloning and GitHub push. Check with git --version
bash verify environment
$ python --version Python 3.11.4   $ java -version openjdk version "11.0.20"   $ git --version git version 2.41.0
02
INSTALL
Clone the repo and install Python dependencies
bash clone & install
$ git clone https://github.com/<your-username>/ocsf-log-mapper.git $ cd ocsf-log-mapper $ pip install -r requirements.txt
ℹ️
After install, run a quick test with the included sample logs: python main.py — this processes sample_logs/raw_logs.json by default.

To run with a custom input file and source flag:

bash custom input
$ python main.py --input path/to/your/logs.json --source aws_cloudtrail
03
PUSH TO GITHUB
Initialize git, commit, connect to your remote, and push
⚠️
Create the repo on GitHub first at github.com/new. Name it ocsf-log-mapper and keep it empty — no README, no .gitignore from GitHub.
bash navigate to project
$ cd C:\Users\dipti.s\Desktop\Ocsf_Log_Mapper
bash initialize git
$ git init
bash stage & verify
$ git add .   # Verify — output/ should NOT appear in this list $ git status
bash first commit
$ git commit -m "Initial commit: OCSF Log Mapper with CloudTrail support"
bash connect & push — replace YOUR_USERNAME
$ git remote add origin https://github.com/YOUR_USERNAME/ocsf-log-mapper.git $ git branch -M main $ git push -u origin main
04
VERIFY
After pushing, confirm your repo structure matches exactly

Your GitHub repo should look like this after a successful push:

ocsf-log-mapper/
├── 📄 .gitignore
├── 📄 LICENSE
├── 📄 README.md
├── 📄 requirements.txt
├── 📄 main.py
├── 📁 engine/
│ ├── __init__.py
│ ├── loader.py
│ ├── resolver.py
│ └── mapper.py
├── 📁 config/mappings/
│ └── aws_cloudtrail.yml
└── 📁 sample_logs/
└── raw_logs.json
# output/ is git-ignored — should NOT appear

Check off each item as you verify:

  • ✓ All 5 root files present (.gitignore, LICENSE, README.md, requirements.txt, main.py)
  • ✓ engine/ folder contains __init__.py, loader.py, resolver.py, mapper.py
  • ✓ config/mappings/aws_cloudtrail.yml is present
  • ✓ sample_logs/raw_logs.json is present
  • ✓ output/ directory does NOT appear in the repo
✅
Deployment complete. Your OCSF Log Mapper is live on GitHub and ready to process logs. Head over to the Upload page to start mapping events.
